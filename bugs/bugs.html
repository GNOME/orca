Orca Bug Dependency List

This page contains a list of bugs filed against external components and applications. This allows us to better track which releases of external components that Orca will work with and also the behavior one might experience in the presence of the bug.

  • GNOME Bugs
  • Evolution Bugs
  • Firefox Bugs
  • OpenOffice Bugs

Evolution 2.4.x 322776: Unable to keyboard navigate into message headers pane, if no message selected. Yes No Workaround: press the "arrow down" button after having tabbed to the message list pane
Evolution 2.4.x 318973: Caret Mode broken No No Originally filed as bug #324002, which was closed as a duplicate of this bug.
Workaround: press the "Tab" key to get focus on the initial email address. Then the arrow keys work.
Evolution 2.4.x 324005: Keynav: Apparently unable to navigate to various components in calendar view. No No Workaround: Seems you can't automatically tab to them the first Time. If you hit Tab when the focus is initially on the JESCS button, you will end up in the Day View Pane.

If you then hit Shift-Ctrl-Tab, the focus goes back to the Clear button. If you hit Shift-Ctrl-Tab again, the focus then goes back to the JESCS button.

If you now hit Tab, focus goes to "Category is". Hitting Tab again takes focus to "Any Category". Hitting Tab again takes focus to the Clear button. Hitting Tab again takes focus back into the Day View pane.

Evolution 2.4.x 324104: Keynav: Apparently no keyboard equivalent for editing calendar appt. Not a bug Yes Workaround: Not a bug. You can use Tab to get focus into the day view. Then use the arrow keys to get focus to the time you require. Then hit Esc to get that appointment into text-entry mode, then Control-O to bring up the calendar appointment popup to edit it.
Evolution 2.4.x 324193: Keynav: Keynav: apparently unable to navigate to certain fields on calendar appt. dialog Not a bug Yes Workaround: When the focus is in the description area, press "Ctrl + tab" can move the focus to the "All day event" checkbox. Then press "Tab", the focus is moved to "Start time".
Evolution 2.4.x 324726: Keynav: Keynav: navigation of HTML mail messages is sub-optimal. No No Workaround: None currently known.
Evolution 2.4.x 326649: Keynav: keynav: Navigating the Preferences dialog is inconsistent. Fixed in CVS HEAD No Workaround: None currently known.
